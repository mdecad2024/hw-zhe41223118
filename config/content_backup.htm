近端

在本周第二支教學影片中：

近端維護共有三種方式，HTTPS SSH GitHub CLI，而SSH需綁定登錄檔較為麻煩，所以本支教學影片使用HTTPS，如果想使用SSH做近端維護，可以觀看先前這個網站教學步驟。

首先：

1.確認要克隆的作業倉儲，複製HTTPS倉儲網址

2.至命令指令，移動到tmp，輸入git clone 網址 (如果後方沒有輸入其他將默認克隆檔名稱)

3.至倉儲，點擊右上方頭像至設定，滑至最下方左側Developer Settings(開發者設定)，Personal access tokens下的tokens (classic)，此設定彈性少，但較為簡單，之後右上點開選擇New personal access token (classic)

4.repo需勾選，之後就能新建tokens，複製tokens，需將其放置克隆好的倉儲目錄的git.下的config裡

5.將tokens放置個人倉儲連結的//後+@

做完以上步驟就能在近端上維護個人網站。

最後網站內容上傳，會因為還未綁定身分而無法上傳，就依序的輸入回答出現問題，之後就能正常地在近端維護網站了。
